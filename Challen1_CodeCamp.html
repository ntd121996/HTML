Using Basic Git Command


.

    When you need init repository:

  • git init
  • git remote add origin [remote repository url]
  • git push -u origin [branch]

    • Push upstream local branch to remote branch

    When you have exist repository:

  • git clone origin [remote repository url]

    • When you need to change remote repository exist repository:

  • git remote git remote set-url origin [remote repository url]

    • Global config user and id:

  • git conflig --global user.name = "Your Name"
  • git conflig --global user.email = "YourName@gmail.com"

    • Show Log:

  • git log

    • Show Log Graphic With One Line and All Branch:

  • git log --graph --decorate --online --all

    • When You Commit and Realized Need Make Small Change:

  • git commit -amend --no-edit

    • About using this command, your history is not add more commit. You could also commit and keep history by using git rebase -i then stash them. Warning: You Shouldn't commit -amend when have been push on remote Branch should change it at local branch

    When You Need Undo Your Change To A file:

  • git log

    • git log to find commit hash

  • git checkout [saved hash] -- path/to/file

    • saved hash is a commit you want to checkout

  • git commit -m "wow, you dont have to copy and paste

    • Git Reset Command:

  • git reset--hard origin branch

    • every thing change in your working directory and staged(index) is change and HEAD is your branch now

  • git reset--soft origin branch

    • HEAD is your branch now but woking directory and staged(index) is not change

  • git reset --mixed origin branch (also git reset origin branch)

    • HEAD is your branch now,staged(index) is change and left in working directory, working directory is not change

    Conclusion:
    --hard: staged(index), working directory change
    --mixed(default): staged(index) change, working directory unchange
    --soft(default): staged(index) unchange, working directory unchange only HEAD change
